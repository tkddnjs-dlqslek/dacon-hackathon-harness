# 평가 기준 체크리스트

대회 채점 기준을 구현 판단의 근거로 사용한다.

---

## 1. 범용성 (25점) — 최우선
다양한 투자 데이터를 넣어도 대시보드가 동작해야 한다.

- [ ] 다른 ETF 티커를 넣어도 동일한 분석 흐름이 작동하는가?
- [ ] 주가 데이터 vs ETF 데이터 vs 포트폴리오 데이터 등 구조가 달라도 대응 가능한가?
- [ ] Skills.md 규칙만 수정하면 새로운 데이터 유형에 대응 가능한가?
- [ ] 하드코딩된 티커/섹터 없이 데이터 기반으로 동적 구성되는가?

## 2. Skills.md 설계 (25점) — 최우선
분석 규칙이 명확하고 체계적이어야 한다.

- [ ] 데이터 분석 규칙이 구체적이고 모호하지 않은가?
- [ ] 시각화 선택 기준이 데이터 특성(시계열/분포/비교 등)에 따라 분기되는가?
- [ ] 인사이트 생성 규칙이 조건 → 메시지 형태로 정의되어 있는가?
- [ ] 여러 개의 Skills.md로 역할 분리가 잘 되어 있는가?
- [ ] Skills.md만 읽어도 대시보드 로직을 이해할 수 있는가?

## 3. 대시보드 자동 생성 (25점) — 최우선
데이터 → 분석 → 시각화까지 자동으로 동작해야 한다.

- [ ] 데이터 입력 시 자동으로 분석이 시작되는가?
- [ ] 차트 유형이 데이터에 맞게 자동 선택되는가?
- [ ] 인사이트가 자동 생성되어 표시되는가?
- [ ] UI가 깔끔하고 완성도 있는가? (레이아웃, 색상, 반응형)
- [ ] 인터랙션 (필터, 기간 선택, 섹터 전환 등)이 동작하는가?

## 4. 바이브코딩 활용 (15점)
Skills.md가 실제 코드 생성의 근거로 작동해야 한다.

- [ ] Skills.md 규칙과 구현 코드 간 1:1 대응이 명확한가?
- [ ] 수동 하드코딩 최소화 — 규칙 변경 시 코드도 자동 대응 가능한 구조인가?
- [ ] Claude Code가 Skills.md를 읽고 구현할 수 있는 수준으로 작성되었는가?

## 5. 실용성 & 창의성 (10점)
실제 투자자에게 유용해야 한다.

- [ ] 실제 투자 의사결정에 도움이 되는 정보를 제공하는가?
- [ ] 차별화 기능: 섹터별 상관관계? 리밸런싱 시뮬레이션? 위험 분석?
- [ ] 사용자 경험: 직관적인 네비게이션, 빠른 로딩, 모바일 대응

---

## 판단 기준
구현 우선순위 결정 시: 범용성 = Skills.md = 대시보드 자동생성 > 바이브코딩 > 실용성
시간 부족 시 범용성·Skills·자동생성에 집중하고, 실용성·창의성은 보너스로 처리.
