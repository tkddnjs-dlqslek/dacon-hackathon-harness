
# 프로젝트 리포트: 멀티 에셋 투자 분석 대시보드

> **한 줄 요약**: "주식·채권·달러·금·비트코인·코스피 중에 뭘 사야 하지?"라는 질문에, **6가지 자산 + 600개 종목 + 재무제표 + 한국어 자연어 질문**으로 답해주는 자동 대시보드를 만들었습니다.

> **5분 둘러보기**: `/demo` 페이지에서 8개 핵심 기능을 단계별로 안내합니다.

---

## 1. 이게 뭐예요?

### 무슨 문제를 풀려고 했나요?

투자를 시작할 때 가장 어려운 것은 **"어디에 돈을 넣어야 할지"** 결정하는 일입니다. 게다가 종류도 너무 많습니다:

- 주식 (애플·삼성전자·SK하이닉스)
- 채권 (미국 국채 금리)
- 환율 (원/달러·유로·엔)
- 원자재 (금·은·원유)
- 암호화폐 (비트코인·이더리움)
- 시장 지수 (코스피·S&P500)

이런 **서로 다른 6가지 종류의 투자 데이터**는 각각 보는 방법도, 분석 기준도 다릅니다. 주식은 "주가가 얼마나 올랐냐"를 보지만, 채권은 "금리가 몇 %냐"를 봅니다. 일반 대시보드로는 이걸 한 화면에서 다루기 어렵습니다.

이 프로젝트는 **"규칙 문서(Skills.md)를 한 번 잘 만들어두면, 어떤 종류의 투자 데이터든 자동으로 분석·시각화·해석이 되는 시스템"**을 만드는 게 목표입니다.

### 왜 만들었나요?

데이콘이 주최한 **"투자 데이터를 시각화하라 — Skills 기반 대시보드 설계"** 해커톤 출품작입니다. 마감 5월 14일.

## 2. 핵심 아이디어 — Skills 기반 자동화

### 일반 대시보드 vs 이 프로젝트

**일반 대시보드**: 개발자가 "주식은 이렇게, 채권은 저렇게"를 일일이 코딩합니다. 데이터가 늘어나면 코드도 늘어납니다.

**이 프로젝트**: **Skills.md 라는 규칙 문서**를 먼저 작성하고, 그 규칙대로 코드가 자동으로 동작합니다. 새 자산 종류를 추가하고 싶으면 **규칙을 수정하면 됩니다**.

### Skills 6개 문서

| 파일 | 역할 |
|------|------|
| `MASTER_SKILL.md` | 전체 조립 설명서 + 6개 자산군 등록부 |
| `data-schema.md` | 다른 형식 데이터를 한 표준으로 변환하는 규칙 |
| `data-analysis.md` | 자산 종류별 어떤 숫자를 계산할지 |
| `visualization.md` | 어떤 데이터엔 어떤 그래프를 쓸지 |
| `insight-generation.md` | "수익률 10% 넘으면 칭찬해" 같은 30+ 규칙 |
| `report-layout.md` | 화면을 어떻게 배치할지 |

자산 종류에 따라 **표시되는 지표가 자동으로 달라집니다**:

| 자산군 | 자동 지표 |

| 외환 | 환율 변동률, **52주 범위** |
| 원자재 | 가격, **주식 상관관계**, **인플레 헤지 점수** |
| 암호화폐 | 가격, **BTC 상관관계** |
| 시장 지수 | 가격, **글로벌 상관관계** |

## 5. 새 자산 추가는 얼마나 쉽나요?

이 시스템의 진짜 자랑입니다.

### 시나리오 1: 새 ETF 추가
`scripts/collect.py`에 한 줄만 추가 → 재실행 → 모든 페이지 자동 반영

### 시나리오 2: 사용자가 임의 종목 분석
`/search` 페이지에서 검색 → 즉시 분석 (사전 수집·코드 수정 불필요)

### 시나리오 3: 새 자산 종류 추가 (예: 부동산)
1. `MASTER_SKILL.md` 등록부에 추가
2. `data-analysis.md §3`에 분석 지표 정의
3. 어댑터의 `supportedTypes`에 추가

코드 수정 거의 없이 가능합니다.

## 6. 대시보드 핵심 차별 기능 (8가지)

### 1️⃣ 멀티 에셋 + 한국 주식
6개 자산 클래스 + 한국 코스피 시총 톱20 동시 분석

### 2️⃣ 자산 타입별 자동 분기
채권은 금리(%)로, 외환은 환율로, 주식은 가격으로 — Skills 규칙 기반 자동

### 3️⃣ **다른 형식 데이터 처리** ⭐ NEW
가격 시계열(OHLCV) + 재무제표 단면(snapshot) 두 형식을 같은 시스템에서 처리. 어댑터 패턴의 진가.

### 4️⃣ 실시간 종목 검색
사전 수집 없이 어떤 종목이든 즉시 분석 (yahoo-finance2 어댑터)

### 5️⃣ 자연어 질문 (한국어)
"기술주 중 변동성 가장 낮은 5개" 같은 한국어 질문 즉시 응답 — 외부 API 키 불필요

### 6️⃣ 자동 시장 리포트
시장 레짐 판단 + TOP5 상승/하락 + 크로스 에셋 분석 자동 생성

### 7️⃣ 효율적 프론티어
1000개 몬테카를로 시뮬레이션으로 최적 포트폴리오 자동 추천

### 8️⃣ 포트폴리오 URL 공유
슬라이더 비중을 URL로 인코딩 → 카톡 링크 공유

## 부록 B: 용어집

| 용어 | 뜻 |
|------|------|
| ETF | 여러 주식을 묶어 파는 상품 |
| 자산 클래스 | 큰 분류 (주식/채권/외환/원자재/암호화폐/지수) |
| 수익률 | 가격이 얼마나 올랐는지 (%) |
| 변동성 | 가격이 얼마나 들쭉날쭉한지 |
| 샤프 비율 | 위험 1단위당 수익이 얼마나 났는지 |
| MDD (최대 낙폭) | 고점 대비 최악으로 얼마나 떨어졌는지 |
| 베타 | 시장 1% 움직일 때 이 종목이 몇 % 움직이는지 |
| 상관관계 | 두 자산이 얼마나 같이 움직이는지 (-1 ~ +1) |
| Yield (금리) | 채권 만기 수익률 (가격이 아니라 %) |
| bps (베이시스 포인트) | 1 bps = 0.01% |
| 환율 | 한 통화를 다른 통화로 바꿀 때 비율 |
| 인플레이션 헤지 | 물가 상승으로부터 가치를 지키는 효과 |
| 리밸런싱 | 비중이 흐트러지면 다시 맞춰주는 것 |
| 백테스트 | "과거에 이렇게 투자했더라면?" 시뮬레이션 |
| 효율적 프론티어 | 위험 대비 수익이 가장 좋은 포트폴리오 조합 |
| 몬테카를로 시뮬레이션 | 랜덤하게 1000번 시도해서 최적 찾기 |
| S&P 500 | 미국 대표 500개 회사 묶음 |
| 유니버스 (Universe) | "분석 대상 종목 풀(pool)" |
| 어댑터 | 다른 형식 데이터를 통일 형식으로 변환하는 모듈 |
| 자연어 쿼리 (NLQ) | 한국어 질문을 코드 없이 분석으로 변환 |
| 시장 레짐 | 위험 자산 선호(리스크 온) vs 안전 자산 선호(리스크 오프) |
| 시계열 (Time Series) | 시간 순서대로 기록된 데이터 (예: 일별 주가) |
| 단면 (Snapshot) | 특정 시점의 한 컷 데이터 (예: 오늘 기준 PER) |
| PER (주가수익비율) | 주가 ÷ 주당순이익. 낮을수록 저평가 |
| PBR (주가순자산비율) | 주가 ÷ 주당순자산. 낮을수록 저평가 |
| ROE (자기자본이익률) | 자기자본 대비 얼마나 이익을 냈는지. 높을수록 효율적 |
| EPS (주당순이익) | 주식 1주당 회사가 번 돈 |
| 시가총액 | 발행 주식 수 × 주가. 회사의 시장 가치 |
| 가치주 | PER, PBR이 낮은 저평가 종목 |
| 성장주 | 매출/이익 성장률이 높은 종목 |
| 배당주 | 정기 배당이 많은 종목 |
