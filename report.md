# 프로젝트 리포트: 멀티 에셋 투자 분석 대시보드

> **한 줄 요약**: "주식·채권·달러·금·비트코인 중에 뭘 사야 하지?"라는 질문에, 6가지 자산을 한 화면에서 비교해 답해주는 자동 대시보드를 만들었습니다.

---

## 1. 이게 뭐예요?

### 무슨 문제를 풀려고 했나요?

투자를 시작할 때 가장 어려운 것은 **"어디에 돈을 넣어야 할지"** 결정하는 일입니다. 게다가 종류도 너무 많습니다:

- 주식 (애플·삼성)
- 채권 (국채 금리)
- 환율 (달러·유로·엔)
- 원자재 (금·은·원유)
- 암호화폐 (비트코인·이더리움)
- 시장 지수 (코스피·S&P500)

이런 **서로 다른 6가지 종류의 투자 데이터**는 각각 보는 방법도, 분석 기준도 다릅니다. 주식은 "주가가 얼마나 올랐냐"를 보지만, 채권은 "금리가 몇 %냐"를 봅니다. 달러는 "환율이 얼마냐"를 보고요. 일반 대시보드로는 이걸 한 화면에서 다루기 어렵습니다.

이 프로젝트는 **"규칙 문서(Skills.md)를 한 번 잘 만들어두면, 어떤 종류의 투자 데이터든 자동으로 분석·시각화·해석이 되는 시스템"**을 만드는 게 목표입니다.

### 왜 만들었나요?

데이콘이 주최한 **"투자 데이터를 시각화하라 — Skills 기반 대시보드 설계"** 해커톤 출품작입니다. 마감 5월 14일.

---

## 2. 핵심 아이디어 — Skills 기반 자동화

### 일반 대시보드 vs 이 프로젝트

**일반 대시보드**: 개발자가 "주식은 이렇게, 채권은 저렇게"를 일일이 코딩합니다. 데이터가 늘어나면 코드도 늘어납니다.

**이 프로젝트**: **Skills.md 라는 규칙 문서**를 먼저 작성하고, 그 규칙대로 코드가 자동으로 동작합니다.

```
Skills.md 규칙 (사람이 작성)
       ↓
"채권은 yield 단위로 표시해라"
"주식은 가격으로, 변동성과 베타도 같이 보여줘라"
"암호화폐는 변동성이 높으니 임계값을 80%로 잡아라"
       ↓
코드가 규칙을 읽고 자동 동작
```

새 자산 종류를 추가하고 싶으면 **규칙을 수정하면 됩니다**. 코드를 다시 짤 필요가 없습니다.

### Skills 6개 문서

| 파일 | 역할 | 비유 |
|------|------|------|
| `MASTER_SKILL.md` | 전체 조립 설명서 + 6개 자산군 등록부 | 책의 목차 |
| `data-schema.md` | 다른 형식의 데이터를 한 가지 표준으로 변환하는 규칙 | 변환 어댑터 |
| `data-analysis.md` | 자산 종류별로 어떤 숫자를 계산할지 | 수학 공식집 |
| `visualization.md` | 어떤 데이터엔 어떤 그래프를 쓸지 | 그래프 가이드 |
| `insight-generation.md` | "수익률 10% 넘으면 칭찬해" 같은 30개 규칙 | 평론가 매뉴얼 |
| `report-layout.md` | 화면을 어떻게 배치할지 | 신문 편집 가이드 |

## 4. 무엇을 보여주나요?

### 📊 메인 대시보드 (`/`)

**"전체 시장이 지금 어떻게 돌아가는지 한눈에"**

- **자산 클래스 필터**: 6개 자산군 중 보고 싶은 것만 토글로 켜고 끄기
- **자산 클래스별 평균 성과 막대 그래프**: 어느 자산군이 잘 됐는지 한눈에
- **크로스 에셋 누적 수익률 그래프**: 6개 자산군의 대표 종목이 시간에 따라 어떻게 움직였는지
- **크로스 에셋 상관관계 격자**: "주식이 오르면 금도 오르나? 채권은?" 같은 질문에 답
- **30개 자산 통합 테이블**: 자산군 색상으로 구분된 전체 자산 성과
- **자동 인사이트**: "주식이 가장 좋았다", "주식과 채권의 상관관계가 변하고 있다" 같은 메시지 자동 생성
- **기간 필터**: 1개월~전체 버튼으로 모든 숫자가 다시 계산

### 🔍 자산 클래스별 깊이 분석 (`/asset-class/[종류]`) ⭐ NEW

**"이 자산군, 자세히 보고 싶어요"**

여기가 이 프로젝트의 **핵심 차별점**입니다. 자산 종류에 따라 **표시되는 지표가 자동으로 달라집니다**:

| 자산군 | 자동으로 표시되는 지표 |
|--------|---------------------|
| 주식/ETF | 가격, 수익률, 변동성, **샤프 비율**, **베타(시장 민감도)** |
| 채권 | **현재 금리(%)**, **금리 변화(bps)**, 변동성 |
| 외환 | 환율 변동률, 변동성, **52주 변동 범위** |
| 원자재 | 가격, 변동성, **주식과의 상관관계**, **인플레이션 헤지 점수** |
| 암호화폐 | 가격, 변동성, **비트코인과의 상관관계**, **주식과의 상관관계** |
| 시장 지수 | 가격, 수익률, **글로벌 상관관계** |

**왜 자동인가?** Skills의 `data-analysis.md §3`에 자산 종류별 규칙이 정의되어 있고, 페이지가 그 규칙을 읽어서 적용합니다. 채권은 가격이 아니라 금리로 보여줘야 한다는 것까지 규칙에 적혀 있어요.

### 🏛️ 섹터 상세 (`/sector/[산업]`)

**"기술주 안에 어떤 회사가 있는지"**

- 그 산업의 대표 ETF + 상위 5개 회사 비교
- 회사별 비중·수익률·변동성 표
- 그 산업 전용 인사이트

### ⚙️ 포트폴리오 만들기 (`/portfolio`)

**"내 마음대로 비중을 섞어서 결과 보기"**

- 11개 ETF 슬라이더 (합계 100% 자동 유지)
- 비중 바꿀 때마다 실시간 백테스트 ("지난 2년간 이렇게 투자했으면 얼마였을까?")
- S&P 500과 비교
- 리밸런싱 시뮬레이션 (정기적으로 비중 맞추는 게 나은지 vs 그냥 두는 게 나은지)

### 🆚 ETF vs 직접 투자 비교 (`/compare`)

**"이 산업에 ETF를 살까, 아니면 직접 종목을 고를까?"**

- 섹터 선택 → 상위 3/5/10개 종목 직접 투자한 결과와 ETF 비교
- 누적 수익률, 변동성, 비용까지 모두 비교
- ETF의 운용보수 vs 직접 투자의 거래 수수료, 결국 어느 게 더 비싼지

## 5. 새 자산 추가는 얼마나 쉽나요?

이 시스템의 진짜 자랑입니다.

### 시나리오 1: 새 ETF 추가
`scripts/collect.py`에 한 줄만 추가:
```python
ASSETS["equity_etf"]["NEW_TICKER"] = {"name": "...", "sector": "..."}
```
재실행하면 끝. 자동으로 모든 페이지에 반영됩니다.

### 시나리오 2: 새 데이터 소스 추가 (CSV 파일 등)
`lib/adapters/` 폴더에 어댑터 한 개만 구현:
```typescript
export const myAdapter: DataAdapter = {
  name: "my-source",
  supportedTypes: ["equity_etf", "bond"],
  async fetch(tickers, period) { ... },
  validate(asset) { ... },
};
```
규칙은 Skills의 `data-schema.md §2`에 정의되어 있어 그것만 따르면 됩니다.

### 시나리오 3: 새 자산 종류 추가 (예: 옵션, 부동산)
1. `MASTER_SKILL.md` 등록부에 추가
2. `data-analysis.md §3`에 분석 지표 정의
3. 어댑터의 `supportedTypes`에 추가

코드 수정 거의 없이 가능합니다.

## 부록 A: 용어집

| 용어 | 뜻 |
|------|------|
| ETF | 여러 주식을 묶어 파는 상품. 분산 투자가 자동으로 됨 |
| 섹터 | 산업 분류 (기술, 금융, 에너지 등) |
| 자산 클래스 | 큰 분류 (주식, 채권, 외환, 원자재, 암호화폐, 지수) |
| 수익률 | 가격이 얼마나 올랐는지 (%) |
| 변동성 | 가격이 얼마나 들쭉날쭉한지. 클수록 위험 |
| 샤프 비율 | 위험 1단위당 수익이 얼마나 났는지. 클수록 효율적 |
| MDD (최대 낙폭) | 고점 대비 최악으로 얼마나 떨어졌는지 |
| 베타 | 시장 전체가 1% 움직일 때 이 종목이 몇 % 움직이는지 |
| 상관관계 | 두 자산이 얼마나 같이 움직이는지 (-1 ~ +1) |
| Yield (금리) | 채권의 만기 수익률. 가격이 아니라 % 단위 |
| bps (베이시스 포인트) | 1 bps = 0.01%. 금리 변화를 표현할 때 씀 |
| 환율 | 한 통화를 다른 통화로 바꿀 때의 비율 |
| 인플레이션 헤지 | 물가 상승으로부터 가치를 지키는 효과 |
| 리밸런싱 | 비중이 흐트러지면 다시 맞춰주는 작업 |
| 백테스트 | "과거에 이렇게 투자했더라면?" 시뮬레이션 |
| 운용보수 | ETF 회사가 가져가는 연 수수료 |
| S&P 500 | 미국 대표 500개 회사 묶음. 시장 평균의 기준점 |
| 어댑터 | 다른 형식의 데이터를 통일된 형식으로 바꿔주는 모듈 |
