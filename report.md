# 프로젝트 리포트: 섹터 ETF 투자 분석 대시보드

> **한 줄 요약**: "어떤 산업에 투자하는 게 좋을까?"라는 질문에 답해주는 자동 대시보드를 만들었습니다.

---

## 1. 이게 뭐예요?

### 무슨 문제를 풀려고 했나요?

주식 투자를 시작할 때 가장 어려운 것은 **"어디에 돈을 넣어야 할지"** 결정하는 일입니다. 개별 회사 주식(예: 애플, 테슬라)을 하나씩 사는 건 어렵고 위험해서, 많은 사람들이 **ETF**라는 상품을 씁니다.

**ETF가 뭐냐면?** "여러 회사 주식을 묶어서 한 번에 사는 상품"입니다. 예를 들어 "기술주 ETF" 하나를 사면 애플·마이크로소프트·엔비디아 같은 기술 회사 수십 개에 한 번에 투자한 효과가 납니다. 장바구니 같은 거라고 생각하시면 됩니다.

문제는 ETF도 종류가 너무 많다는 것입니다:
- 기술주 ETF, 에너지주 ETF, 헬스케어주 ETF, ...
- 어떤 게 수익률이 좋았는지?
- 위험은 얼마나 큰지?
- 여러 개를 어떻게 섞어야 좋은지?
- ETF를 사는 것보다 차라리 직접 종목을 고르는 게 나을까?

이런 질문에 **데이터로 답해주는 자동 대시보드**가 이 프로젝트입니다.

### 왜 만들었나요?

데이콘이라는 데이터 분석 플랫폼에서 주최한 **"투자 데이터를 시각화하라 — Skills 기반 대시보드 설계"** 해커톤에 출품할 작품입니다. 마감일은 5월 14일입니다.

## 2. 무엇을 보여주나요? (4개 페이지)

### 📊 페이지 1: 메인 대시보드 (`/`)

**"전체 시장이 지금 어떻게 돌아가는지 한눈에"**

- **인사이트 카드**: 자동으로 "이 ETF가 최근 한 달 15% 올랐어요!" 같은 메시지를 만들어서 보여줍니다. 빨강(경고)/노랑(주의)/초록(긍정)/파랑(정보) 4단계로 분류합니다.
- **요약 숫자 4개**: 평균 수익률, 변동성(가격이 얼마나 흔들리는지), 샤프 비율(위험 대비 수익이 얼마나 좋은지), 최대 낙폭(최악의 경우 얼마나 떨어졌는지)
- **섹터 비중 도넛**: 11개 산업이 어떤 비중으로 들어있는지
- **누적 수익률 그래프**: 11개 ETF가 시간에 따라 얼마나 올랐는지 한 그래프에
- **섹터 랭킹 막대**: 어느 산업이 잘나가고 어느 산업이 부진한지
- **성과 표**: 모든 ETF의 수익률, 위험도를 표로 정리
- **상관관계 격자**: 어떤 산업끼리 같이 움직이는지 색깔로 표현 (분산 투자할 때 중요)
- **기간 필터 ⭐**: 1개월/3개월/6개월/1년/전체 버튼을 누르면 모든 숫자와 그래프가 그 기간 기준으로 다시 계산됩니다

### 🔍 페이지 2: 섹터 상세 (`/sector/[산업명]`)

**"이 산업, 자세히 보고 싶어요"**

특정 산업을 선택하면:
- 그 산업의 핵심 지표 4개
- 그 산업에 어떤 회사들이 들어있는지 (예: 기술주 ETF에는 엔비디아, 애플, 마이크로소프트 등)
- ETF와 그 안의 회사들이 시간에 따라 어떻게 움직였는지 비교 그래프
- 회사별 비중·수익률·변동성 표
- 그 산업 전용 인사이트

### ⚙️ 페이지 3: 포트폴리오 만들기 (`/portfolio`)

**"내 마음대로 섞어서 결과를 보고 싶어요"**

- **슬라이더 11개**: 각 산업에 몇 %씩 투자할지 직접 조절
- **자동 100% 유지**: 하나를 올리면 나머지가 비례적으로 자동 조정
- **즉시 백테스트**: 비중을 바꿀 때마다 "지난 2년간 이렇게 투자했으면 얼마가 됐을까?"를 실시간으로 계산해서 그래프로 보여줌
- **벤치마크 비교**: S&P 500(미국 대표 지수)과 비교
- **리밸런싱 시뮬레이션**: 정기적으로 비중을 다시 맞추는 게 좋은지, 그냥 두는 게 좋은지를 비교 (월간/분기/반기/연간 옵션)

### 🆚 페이지 4: ETF vs 직접 투자 비교 (`/compare`) — 차별화 기능

**"이 산업에 투자하는데, ETF를 살까 아니면 직접 종목을 고를까?"**

이게 이 프로젝트의 **핵심 차별화 기능**입니다.

- **섹터 선택**: 11개 산업 중 하나
- **종목 수**: 그 산업의 상위 3개/5개/10개 종목을 고를지
- **리밸런싱**: 정기적으로 비중을 맞출지
- **결과**: ETF를 산 경우 vs 개별 종목을 직접 산 경우를 나란히 비교
  - 누적 수익률 그래프 (실선 vs 점선)
  - 위험 지표 비교 (변동성, 샤프, 최대 낙폭)
  - **누적 비용 비교**: ETF는 운용보수가 있고, 직접 투자는 거래 수수료가 있는데 어느 쪽이 결국 더 비싼지
  - 종목별 기여도 표

이걸 보면 **"ETF가 정말 그 가치를 하는지"**를 정량적으로 판단할 수 있습니다.

## 3. 뭐가 특별한가요? — Skills 기반 자동 생성

### 일반 대시보드 vs 이 프로젝트

**일반 대시보드**: 개발자가 "이 데이터는 막대 그래프로, 저 데이터는 선 그래프로"를 일일이 코딩합니다. 새 데이터가 들어오면 다시 코딩해야 합니다.

**이 프로젝트**: **Skills.md 라는 규칙 문서**를 먼저 작성하고, 그 규칙대로 코드가 자동으로 동작합니다. 새 ETF를 추가해도 규칙이 알아서 처리합니다.

### Skills 5개 문서

| 파일 | 역할 | 비유 |
|------|------|------|
| `MASTER_SKILL.md` | 전체 흐름 정리 | 책의 목차 |
| `data-analysis.md` | 숫자 계산 규칙 | 수학 공식집 |
| `visualization.md` | 차트 선택 규칙 | "이런 데이터엔 이런 그래프" 가이드 |
| `insight-generation.md` | 메시지 생성 규칙 | "수익률이 10% 넘으면 칭찬해" 같은 30개 규칙 |
| `report-layout.md` | 화면 배치 규칙 | 신문 편집 가이드 |

규칙 문서를 수정하면 코드도 따라 바뀝니다. 예를 들어 "수익률 10% 이상 → 긍정 메시지"를 "15% 이상"으로 바꾸고 싶으면 문서 한 줄만 수정하면 됩니다.

## 4. 데이터는 어디서 가져오나요?

**yfinance**라는 무료 라이브러리 하나만 사용합니다.

- **API 키 필요 없음**: 심사하시는 분이 별도 등록 없이 바로 실행 가능
- **빠름**: 11개 ETF의 2년치 가격을 약 1초 만에 다운로드
- **풍부함**: 가격뿐 아니라 ETF 구성 종목, 운용보수, 무위험 수익률까지 전부 가져옴

심사자가 처음부터 재현하려면 이 명령어 두 줄이면 끝납니다:
```bash
pip install yfinance
python scripts/collect.py
npm install && npm run dev
```

## 부록: 용어집

| 용어 | 뜻 |
|------|------|
| ETF | 여러 주식을 묶어 파는 상품. 분산 투자가 자동으로 됨 |
| 섹터 | 산업 분류 (기술, 금융, 에너지 등) |
| 수익률 | 가격이 얼마나 올랐는지 (%) |
| 변동성 | 가격이 얼마나 들쭉날쭉한지. 클수록 위험 |
| 샤프 비율 | 위험 1단위당 수익이 얼마나 났는지. 클수록 효율적 |
| MDD (최대 낙폭) | 고점 대비 최악으로 얼마나 떨어졌는지 |
| 베타 | 시장 전체가 1% 움직일 때 이 종목이 몇 % 움직이는지 |
| 상관관계 | 두 종목이 얼마나 같이 움직이는지 (-1 ~ +1) |
| 리밸런싱 | 비중이 흐트러지면 다시 맞춰주는 작업 |
| 백테스트 | "과거에 이렇게 투자했더라면 어땠을까?" 시뮬레이션 |
| 운용보수 | ETF 회사가 가져가는 연 수수료 (보통 0.08% 정도) |
| S&P 500 | 미국 대표 500개 회사 묶음. 시장 평균의 기준점 |
